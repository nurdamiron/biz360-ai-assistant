# Инструкции:
1. Создайте специализированный чеклист для код-ревью, который фокусируется на специфических аспектах указанного типа файлов.

2. Чеклист должен содержать проверки, которые особенно важны для данного типа файлов:

   {{#if fileType == "javascript"}}
   - **Структура и организация** - проверки модульности, архитектуры кода
   - **Типизация и проверка данных** - использование типов, валидация входных данных
   - **Асинхронность** - правильная работа с Promise, async/await, обработка ошибок
   - **Управление состоянием** - хранение и изменение состояния
   - **Производительность JS** - эффективность алгоритмов, оптимизации, memory leaks
   - **Безопасность** - защита от XSS, защита данных, инъекций
   - **Тестирование JS** - юнит-тесты, интеграционные тесты, мокинг
   {{/if}}

   {{#if fileType == "css"}}
   - **Организация CSS** - структура, именование классов, селекторы
   - **Отзывчивый дизайн** - медиа-запросы, mobile-first подход
   - **Производительность CSS** - избегание слишком сложных селекторов, оптимизация рендеринга
   - **Кроссбраузерность** - поддержка разных браузеров, префиксы
   - **Методология** - соответствие BEM, SMACSS, OOCSS или другим методологиям
   - **Доступность в CSS** - контрастность, размер шрифта, фокус
   {{/if}}

   {{#if fileType == "html"}}
   - **Семантика** - использование правильных HTML5 элементов
   - **Доступность** - ARIA атрибуты, alt теги, структура заголовков
   - **SEO** - мета-теги, структура данных
   - **Производительность HTML** - оптимизация загрузки, критический путь рендеринга
   - **Валидность** - соответствие стандартам W3C
   - **Совместимость** - работа в разных браузерах
   {{/if}}

   {{#if fileType == "config"}}
   - **Безопасность конфигурации** - проверка секретов, токенов
   - **Согласованность** - соответствие конфигурации разным окружениям
   - **Структура** - организация конфигурационных параметров
   - **Валидация** - проверка формата и значений
   - **Документация** - комментарии и пояснения к параметрам
   {{/if}}

   {{#if fileType == "generic"}}
   - **Соответствие стандартам** - следование общепринятым практикам для данного типа файлов
   - **Документация** - наличие комментариев и пояснений
   - **Качество кода** - читаемость, поддерживаемость
   - **Интеграция** - взаимодействие с другими частями системы
   {{/if}}

3. Для каждого пункта:
   - Сформулируйте четкий вопрос, на который можно ответить "Да/Нет/Не применимо"
   - При необходимости добавьте краткое объяснение, почему это важно
   - Где возможно, сделайте пункты максимально конкретными для представленных файлов

4. Чеклист должен быть подробным и исчерпывающим, но не менее 15 и не более 30 пунктов.

Ответ должен быть в формате Markdown с чекбоксами, заголовками и другими элементами форматирования для улучшения читаемости.
