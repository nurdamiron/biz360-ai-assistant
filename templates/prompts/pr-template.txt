Создайте шаблон для Pull Request на основе предоставленной информации.

# Основная информация
{{#if taskId}}
- ID задачи: {{taskId}}
{{/if}}
{{#if taskTitle}}
- Название задачи: {{taskTitle}}
{{/if}}
{{#if repositoryUrl}}
- URL репозитория: {{repositoryUrl}}
{{/if}}

# Инструкции:
1. Создайте стандартизированный шаблон Pull Request, который может быть использован как основа для будущих PR.
2. Шаблон должен включать следующие разделы:
   - Заголовок PR (с шаблоном заполнения)
   - Описание изменений (с местом для заполнения)
   - Связанные задачи (с автоматическим заполнением на основе указанного taskId, если есть)
   - Контрольный список (чеклист) изменений для автора
   - Инструкции для ревьюеров
   - Раздел тестирования
   - Дополнительные замечания

3. Используйте Markdown для форматирования шаблона.
4. Включите подсказки в виде комментариев, объясняющие, как заполнять каждый раздел.
5. Если taskId или taskTitle указаны, автоматически включите их в шаблон.

Ответ должен содержать только текст шаблона в формате Markdown, готовый для использования в GitHub/GitLab без дополнительной обработки.
