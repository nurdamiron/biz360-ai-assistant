# Инструкции:
1. Преобразуйте предложенное изменение в структурированную задачу для разработчиков.
2. Создайте чёткое и конкретное название задачи, которое ясно описывает, что нужно сделать.
3. Разработайте подробное описание, которое включает:
   - Контекст и обоснование (почему это изменение важно)
   - Конкретные требования и критерии приемки
   - Ограничения или соображения, которые следует учесть
   - Предлагаемое решение (если оно известно)
   - Информацию о том, сколько пользователей запросили это изменение

4. Верните результат в формате JSON:
```json
{
  "title": "<название задачи>",
  "description": "<подробное описание задачи в формате Markdown>",
  "acceptanceCriteria": [
    "<критерий 1>",
    "<критерий 2>",
    "..."
  ],
  "estimatedEffort": "<low|medium|high>",
  "suggestedPriority": "<high|medium|low>"
}
```

5. При составлении описания:
   - Используйте Markdown для форматирования
   - Структурируйте описание с заголовками
   - Будьте конкретны и избегайте двусмысленностей
   - Используйте информацию о количестве упоминаний для определения важности

Верните ответ только в формате JSON, без дополнительных комментариев.
