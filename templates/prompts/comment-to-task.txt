# Инструкции:
1. На основе предоставленных комментариев к коду создайте задачу, которая охватывает все выявленные проблемы и предложения.
2. Приоритизируйте наиболее серьезные проблемы.
3. Объедините похожие проблемы в одну задачу.
4. Сформулируйте чёткое и конкретное название задачи.
5. Создайте детальное описание, которое:
   - Указывает на конкретные проблемы и их расположение в коде
   - Объясняет, почему эти проблемы важны
   - Предлагает возможные пути решения (если они есть в комментариях)

6. Верните результат в JSON формате:
```json
{
  "title": "<краткое, но информативное название задачи>",
  "description": "<подробное описание задачи в формате Markdown>",
  "priority": "<high|medium|low>",
  "estimatedEffort": "<low|medium|high>",
  "relatedLines": [<номера строк>]
}
```

7. Определите приоритет задачи на основе серьезности проблем:
   - "high" - если есть хотя бы одна критическая проблема
   - "medium" - если есть серьезные, но не критические проблемы
   - "low" - если есть только незначительные проблемы

Верните ответ только в формате JSON, без дополнительных комментариев.
