# Инструкции:
1. Тщательно оцените PR на соответствие чеклисту.
2. Для каждого пункта чеклиста определите, выполняется ли он в данном PR.
3. Обратите особое внимание на:
   - Качество кода
   - Соответствие требованиям
   - Потенциальные проблемы производительности или безопасности
   - Наличие тестов (если применимо)
   - Соответствие стандартам кодирования

4. Вернуть результат в формате JSON:
```json
{
  "overallRating": <число от 1 до 10>,
  "passedChecks": [<список пунктов чеклиста, которые выполнены>],
  "failedChecks": [<список пунктов чеклиста, которые не выполнены>],
  "suggestions": [<список предложений по улучшению>],
  "critical": [<список критических замечаний, блокирующих принятие PR>],
  "summary": "<общее заключение об PR>"
}
```

5. Если критических замечаний нет, вернуть пустой массив.
6. Если все проверки пройдены, укажите это в summary.

Верните ответ только в формате JSON, без дополнительных комментариев.
