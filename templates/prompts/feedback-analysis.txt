# Инструкции:
1. Тщательно проанализируйте обратную связь пользователя.
2. Извлеките следующую информацию:
   - Общий тон обратной связи (положительный, отрицательный, нейтральный)
   - Оценка настроения (от -5 до 5, где -5 крайне негативное, 0 нейтральное, 5 крайне положительное)
   - Основные категории, к которым относится обратная связь
   - Конкретные предложения по улучшению, если они есть
   - Конкретные проблемы или жалобы, если они есть
   - Запросы на новые функции, если они есть

3. Предоставьте результат в структурированном JSON формате:
```json
{
  "sentimentScore": <число от -5 до 5>,
  "tone": "<положительный|отрицательный|нейтральный>",
  "categories": [
    "<категория1>", 
    "<категория2>", 
    ...
  ],
  "suggestions": [
    "<предложение1>", 
    "<предложение2>", 
    ...
  ],
  "issues": [
    "<проблема1>", 
    "<проблема2>", 
    ...
  ],
  "featureRequests": [
    "<запрос1>", 
    "<запрос2>", 
    ...
  ],
  "summary": "<краткое резюме обратной связи в одном предложении>"
}
```

4. Если из контекста недостаточно информации для заполнения какого-либо поля, оставьте пустой массив [] для списков или null для значений.

Верните ответ только в формате JSON, без дополнительных комментариев.
