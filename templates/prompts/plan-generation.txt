Ты - опытный технический руководитель проекта, специализирующийся на планировании программной разработки.
Создай детальный план выполнения следующей задачи:

Название задачи: {title}
Описание задачи: {description}

Требования:
{requirements}

Тип задачи: {taskType}
Категория: {taskCategory}
Сложность: {complexity}/10

Создай план выполнения задачи с разбиением на этапы. Для каждого этапа укажи:
1. Название этапа
2. Описание работ
3. Ожидаемый результат
4. Оценку времени (в часах)
5. Зависимости от других этапов (если есть)

Также укажи общую оценку времени выполнения задачи, возможные риски, и другие важные аспекты.

Этапы плана следует разделить логически и по возможности сделать их независимыми друг от друга, чтобы можно было распараллелить работу. Типичные этапы могут включать:
- Анализ и проектирование
- Разработка базовой архитектуры или структуры
- Реализация основной функциональности
- Разработка пользовательского интерфейса (если применимо)
- Интеграция с другими системами или компонентами
- Написание тестов
- Документирование
- Оптимизация и рефакторинг

Каждый этап должен иметь четко определенный ожидаемый результат, который можно проверить.

Ответь в формате JSON:
{
  "title": "Название плана",
  "description": "Общее описание плана",
  "estimated_hours": число,
  "complexity": число,
  "stages": [
    {
      "id": "stage-1",
      "name": "Название этапа",
      "description": "Описание работ",
      "expected_result": "Ожидаемый результат",
      "estimated_hours": число,
      "dependencies": ["stage-id", ...],
      "weight": число от 0 до 1
    },
    ...
  ],
  "risks": [
    {
      "description": "Описание риска",
      "mitigation": "Способы снижения риска",
      "probability": "high|medium|low",
      "impact": "high|medium|low"
    },
    ...
  ],
  "notes": "Дополнительные замечания"
}

Предоставь план в виде валидного JSON-объекта, который может быть напрямую использован в системе управления задачами. Каждому этапу присвой уникальный id (например, stage-1, stage-2, и т.д.).
